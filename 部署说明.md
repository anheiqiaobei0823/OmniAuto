# OmniAuto 部署说明

## 版本

**v0.1.1**

Docker 镜像：`omniauto:v0.1.1`

## 部署步骤

### 1. 解压

```bash
tar xzf OmniAuto-v0.1.1.tar.gz
cd code
```

### 2. 修改 docker-compose.yml

修改以下环境变量：

| 变量 | 说明 |
|------|------|
| OMNIAUTO_ADMIN_PASSWORD | 管理员密码，务必修改 |
| OMNIAUTO_JWT_SECRET | ⚠️ JWT 签名密钥，务必改为随机字符串 |
| OMNIAUTO_REQUIRE_AUTH | 1=开启登录验证，0=关闭 |

### 3. 构建并启动

```bash
docker-compose up -d --build
```

### 4. 验证

```bash
# 检查容器状态
docker ps | grep omniauto

# 测试接口
curl http://localhost:8000/v1/models
# 应返回: {"object":"list","data":[{"id":"OmniAuto",...}]}
```

### 5. 访问管理后台

浏览器打开 `http://服务器IP:8000/OmniAuto`

首次登录：
- 用户名：`admin`
- 密码：你在 `OMNIAUTO_ADMIN_PASSWORD` 中设置的密码

## 部署后配置（在管理后台操作）

1. **设置页** → 填写智能路由模型配置（API 地址、API Key、模型名），可一键导入供应商
2. **供应商与模型** → 添加供应商，拉取模型
3. **路由** → 为每个分类配置模型优先级
4. **API Key** → 生成 Key 供客户端使用

## 端口

- 8000：API + 管理后台

## 数据持久化

- `./data/omniauto.db`：数据库
- `./data/images/`：生成的图片（7天自动清理）
